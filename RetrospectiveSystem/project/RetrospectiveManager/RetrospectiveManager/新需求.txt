一、接口
	接口1：存储测试结果（SN+型号+站位+结果），存储每次的结果
		string InsertData(string sn,string typeNo,string station,string dateTime,string result);
		返回结果：成功-1，失败-0，参数为空

	接口2：查询测试结果，查询时，返回上次最后一条记录
		string SelectData(string sn,string typeNo,string station);
		返回结果：字符串/数组/集合/JSON

	接口3：物料表（最多10种），每种物料存储信息：二维码
		   (上位机录入/可修改)产品型号 + 物料编码；type1+code1+code2+code4;
		   （上位机录入/修改）产品型号 + 物料编码 + 物料名称 + 物料信息(生产厂家)
		   
		   产品：二维码+型号
	接口4： 物料计数表,该产品消耗每种物料的数量，装配完计数，由测试端发送
			string insertMaterialNum(string sn,string typeNo,string materialCode,materialNum);

装配前的站位发送PCB版码（内码），装配之后发送外壳码
物料：外壳+PCB+螺丝+支架+防火栓+线+外箱
装配2之后更新物料，新增

二、存储
	FTP：存储测试数据
		 存储3个站位的测试标准，以文件形式存储，文件中可能存在很多步骤，每个步骤标准值不同，如1-4
		 如何修改？

三、数据库
	1)箱子-产品表
		描述：用外箱存储一定数量的产品，可根据外箱编码查询该箱子中的产品信息，如产品数量、产品型号、产品编码
		表1字段：外箱编码+存储容量
		表2字段：外箱编码+产品编码+产品型号
	2）产品-物料表
		描述：不同产品型号绑定不同的物料，可新增/修改
		表1字段：产品型号 + 物料编码
		表2字段：物料编码 + 物料名称
	3）外箱-物料
		描述：外箱存储物料，记录物料编码（包含多种信息）、物料数量；一种物料一个外箱一个编码
		字段：外箱

四、上位机

（1）产品-物料
	 描述：配置不同产品对应的不同物料
	 功能：
		1）配置产品的物料
		2）录入物料编码等：编辑/修改---产品型号+物料名称+物料编码
（2）外箱-物料：物料编码+物料数量
（3）产品-物料-可修改数量，如外箱中零件数量
（5）成品纸箱：抽查产品时，如有坏的，需要解除绑定，与重新绑定,添加绑定
	 产品完成后，装入箱子时，需记录箱子编码+每个产品的二维码：箱子 + 产品；以便回头查询箱子中的产品，
	 由外箱码  查询 产品，箱子里面有哪些产品，产品的编码；不同箱子，存储不同的产品
	 扫码枪扫描后可设置自动添加到箱子
	 功能：
		外箱的最大存储数量；编辑/修改
		外箱编码；编辑/修改
		添加绑定：扫码枪扫描添加绑定
		重新绑定：扫码枪扫描重新绑定
		解除绑定：抽查产品，不合格时，解除绑定时标记，可查询所有不合格/解除的产品，统计中显示，合格率等等
		查询与显示：根据外箱码查询当前箱子中的产品信息，也可搜索产品查询所属箱子，以及详细信息，如绑定日期，是否合格，是否解除过绑定
	 绑定信息：
		录入-外箱编码+外箱容量+产品编码+产品型号+绑定日期=》保存
		可设置录入保存
