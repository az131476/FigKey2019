一、接口
	接口1类：用户信息表
		调用对象：服务端上位机
	接口2类：产线站位
		调用对象：服务上位机，建议先由上位机配置
	接口3类：产品所属站位
		调用对象：服务上位机，上位机配置型号-站位
	接口4类：产品型号
		调用对象：服务上位机，上位机配置所有型号
	接口5类：存储测试结果（SN+型号+站位+结果），存储每次的结果
		string InsertData(string sn,string typeNo,string station,string dateTime,string result);
		返回结果：成功-1，失败-0，参数为空

	接口6类：查询测试结果，查询时，返回上次最后一条记录
		string SelectData(string sn,string typeNo,string station);
		返回结果：字符串/数组/集合/JSON

	接口7类：物料表（最多10种），每种物料存储信息：二维码
	接口8类：产品型号 + 物料编码；type1+code1+code2+code4;
		   
	接口5类：物料计数表,该产品消耗每种物料的数量，装配完计数，由测试端发送
			string insertMaterialNum(string sn,string typeNo,string materialCode,materialNum);
	接口6类：产品外箱编码+存储容量
		   用外箱存储一定数量的产品，可根据外箱编码查询该箱子中的产品信息，如产品数量、产品型号、产品编码
	接口7类：外箱编码+产品编码+产品型号

装配前的站位发送PCB版码（内码），装配之后发送外壳码
物料：外壳+PCB+螺丝+支架+防火栓+线+外箱
装配2之后更新物料，新增

二、存储
	FTP：存储测试数据
		 存储3个站位的测试标准，以文件形式存储，文件中可能存在很多步骤，每个步骤标准值不同，如1-4
	搭建FTP服务器，客户端自己传输，下载	 
	FTP寄宿IIS

三、数据库
	1）用户信息表
		描述：存储用户信息
		字段：用户名+密码+手机号+电子邮件+头像
	2）产线站位
		描述：配置所有站位
	3）产品-所属站位
		描述：不同型号的产品包含不同的站位流程
	4）产品型号
		描述：配置产品的所有型号
	5)测试结果数据存储
		描述：存储测试结果数据的表
	6）物料表
		描述：配置物料信息的表
		字段：ID+物料编码
	7）产品-物料表
		描述：不同产品型号绑定不同的物料，可新增/修改
		表1字段：产品型号 + 物料编码
	8）物料计数表
		描述：同一产品消耗每种物料的数量，一种物料一个外箱码/物料码；外箱存储物料，
		记录物料编码（包含多种信息）、物料数量；一种物料一个外箱一个编码
		字段：追溯码（外壳码）+型号+ 站位名称+ 物料编码+物料数量
		主键：追溯码+站位名称
	9)箱子-产品表
		描述：用外箱存储一定数量的产品，可根据外箱编码查询该箱子中的产品信息，如产品数量、产品型号、产品编码
		表1字段：外箱ID+外箱编码+存储容量
		表2字段：外箱编码+产品编码+产品型号+绑定日期

四、上位机
（一）配置功能
	1）配置产线站位信息：总共有哪些站位，修改站位，新增站位，删除站位等
	2）配置产品站位：不同型号可能有不同的站位
	3）配置产品型号：配置要管理和生产的所有产品
	4）配置物料信息：配置所有产品可能用到的物料
	5）配置产品物料：不同产品型号用到的物料信息不同
	6）产品打包：打包时需要将产品与箱子绑定
		设置：箱子编码+箱子容量；设置扫码枪扫描后自动绑定
		绑定：箱子编码+产品追溯码
		解除绑定：抽查坏的/不合格的解除绑定
		重新绑定：不合格产品处理后变为合格，可重新添加绑定

（二）查询功能
	1）查询测试结果，按条件查询（设置起始时间）
		按追溯码查询：查询所有记录、查询上一站位记录
		按型号查询：查询所有记录
	2）物料计数
		查询：查询不同产品下，所使用的物料种类和数量（需不需要查询该物料是在哪个站位使用的物料及数量），
			  并显示该物料是在哪个站位使用的，以及具体数量
		编辑/修改：修改物料总数
	3）打包产品
		查询1：根据外箱编码查询箱子中的产品种类、同一产品数量、所有产品数量
		查询2：根据产品（追溯码/型号）查询所属箱子，此情况应用场景为-拿出的产品忘记在哪个箱子存放时，可扫描查询位置

	4）合格率
		查询：根据抽查产品不合格数计算合格率
			显示：由产品分类，统计不同产品的合格率

		查询与显示：根据外箱码查询当前箱子中的产品信息，也可搜索产品查询所属箱子，以及详细信息，如绑定日期，是否合格，是否解除过绑定

备注：所有涉及到编码的都直接用扫码枪扫描
